When we are not sure about the size of a variable at runtime, Rust uses the heap to store values that are growable, 
such as String or Vec. In such cases,  Rust creates a stack frame that contains a pointer, length, and capacity:

Pointer: the address in the heap where the actual data is stored
Length: the current number of elements (e.g., characters in a String)
Capacity: the total amount of space allocated in the heap for future growth

If the data grows beyond the current capacity, Rust will automatically allocate a new block of memory in the heap 
(if contiguous space isn’t available), copy the data over, and update the pointer in the stack frame to point to the new location. ^aXE4auJE
